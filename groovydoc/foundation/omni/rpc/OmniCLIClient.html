class OmniCLIClient
extends foundation.omni.rpc.OmniClient

                  Bitcoin and Omni JSON-RPC client that closely mirrors JSON-RPC API.

                  A JSON-RPC client for Bitcoin/Omni with JVM method names that match the RPC method names and bitcoin-cli method names exactly. (in other words, all lowercase and underscores)

                  We use Groovy defaults to set automatic parameters to null. This is better than using Java varargs because it allows us to specify a unique type for each parameter. We set parameters to null, not the default values in the RPC API because we want the server to choose the defaults, not our client. A null parameter is not sent in the RPC request.

                  deprecated:
                  Use foundation.omni.rpc.OmniClient (or var dynamicClient = omniClient as DynamicRpcMethodFallback if you really want lower-case RPC method names)

                      Get the balance in decimal bitcoins across all accounts or for a particular account. @return

                      Parameters:
                      account - the name of the account to get a balance for or “*” to get the balance for all accounts (the default). The default (primary) account can be specified using “”, which is not the same as specifying “*” for all accounts.
                      minconf - (optional) the minimum number of confirmations an incoming transaction must have before it is counted towards the balance.

                    • Transaction getrawtransaction(Sha256Hash txid)

                      Get the rawtransaction-format data for a transaction.

                      By default, bitcoind only stores complete transaction data for UTXOs and your own transactions, so the RPC may fail on historic transactions unless you use the non-default txindex=1 in your bitcoind startup settings.

                      Note: The verbose param is not supported in this method since we're returning a strongly-typed Transaction object which (in theory) offers the functionality of both the raw bytes and the JSON object that would be returned by the lower-level JSON-RPC API.

                      @return

                      Parameters:
                      txid - the transaction identifier (hash) of the transaction to get.

                    • Coin getreceivedbyaddress(Address address, Integer minconf)

                      Returns the total amount received by the specified address in transactions with at least the indicated number of confirmations.

                      Parameters:
                      address - a Bitcoin address to check. Must be an address belonging to the wallet.
                      minconf - (optional) the minimum number of confirmations a transaction must have before it is counted towards the total. 1 is the default; use 0 to also count unconfirmed transactions.
                      Returns:
                      the total amount received
